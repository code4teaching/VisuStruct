## Deutsch

Diese Version richtet **Marke und technische Identität** an **VisuStruct** aus. **Projektdateien** (`.visustruct` / XML) sind unverändert lesbar; im XML werden Elementtypen weiterhin numerisch gespeichert.

- **Java-Paket:** Quellcode unter **`de.visustruct`** (`control`, `view`, `other`, `struktogrammelemente`) statt `de.kekru.struktogrammeditor`.
- **Maven:** Koordinaten **`de.visustruct:visustruct:1.0.4`** (`artifactId` **visustruct**). Wer das Projekt als Abhängigkeit einbindet, muss **groupId**, **artifactId** und ggf. **version** anpassen. Fat-JAR: **`visustruct-1.0.4.jar`**. Der Release-Workflow liefert zusätzlich **`visustruct.jar`** mit festem Dateinamen.
- **Einstellungen:** Standarddatei **`visustruct.properties`**. Nur noch **`struktogrammeditor.properties`** vorhanden → Werte werden beim Start übernommen; beim nächsten Speichern der Einstellungen schreibt die App **`visustruct.properties`** und entfernt die alte Datei, sofern möglich.
- **macOS:** **Bundle-Identifier** und **UTI** auf **`de.visustruct.*`**. Ggf. **„Öffnen mit“** für `.visustruct` erneut zuordnen. Main-Class: **`de.visustruct.control.Main`**.
- **Upstream:** [kekru/struktogrammeditor](https://github.com/kekru/struktogrammeditor) bleibt in README und Infodialog genannt (MIT).

## English

- Java packages: **`de.visustruct.*`**. Maven: **`de.visustruct:visustruct:1.0.4`**. Main: **`de.visustruct.control.Main`**.
- Settings: **`visustruct.properties`**; migrates from **`struktogrammeditor.properties`** on next save.
- macOS bundle ID / UTI: **`de.visustruct`** — you may need to re-assign **Open With** for `.visustruct`.
- **`.visustruct` / XML** files remain compatible. Upstream **kekru/struktogrammeditor** remains credited.
